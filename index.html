02 — no stars, just clicks

Most feedback isn't a rating. It's a footprint.

Netflix sees a watch, Amazon a buy, Spotify a play — a plain 1, and otherwise silence. Silence isn't "dislike," it's "maybe unseen," so implicit MF trusts the ones and stays soft on the zeros. Toggle cells and watch recommendations surface.

03 — the new update rule

Turn up d for richer factors — the cost stays friendly.

Both panels run the same latent dimension as it climbs automatically. The standard all-pairs cost is dominated by the catalog and balloons; the proposed Gram-trick update grows only like d² and each factor solves independently, in parallel.

[ Bᵀ diag(c²) B + (ρ + λr²) Id ] a = Bᵀ(p ∘ c²) + ρh  →  one d×d solve per user/item.
latent dimension d = 1
10k users · 40k items · 200k interactions

Standard approach

All-pairs cost balloons with the catalog

Comparing every user against every item, per dimension — O(d·|U|·|I|).

Proposed method

Gram-trick keeps it low — and parallel

One reused d×d Gram matrix — O(d²·(|U|+|I|) + d·|S|), solved per factor independently.

04 — share the sum, not the secret

Four companies. One shared total. No one's numbers exposed.

Each domain must add its factor update to a shared average without revealing it. Every pair shares a secret mask: one adds it, the other subtracts it. Broadcasts look like noise, but summed the masks cancel and only the true total survives.

05 — deniability by design

Add just enough noise to make one person invisible.

Even a shared embedding can leak. Differential privacy blurs it so two datasets differing by one user look almost identical. Shrink the budget ε and the curves drift together — the attacker loses the trail, at a measured cost to accuracy.

Indistinguishability

Two neighbouring datasets (differ by one user) → two output distributions.

smaller ε = stronger privacy = more noise (σ ∝ 1/ε)

The trade-off

A jittering sample, the attacker's odds, and the price in accuracy.
