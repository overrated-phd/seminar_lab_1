02 — no stars, just clicks

Most feedback isn't a rating. It's a footprint.

Netflix sees a watch, Amazon a buy, Spotify a play — a plain 1, and otherwise silence. Silence isn't "dislike," it's "maybe unseen," so implicit MF trusts the ones and stays soft on the zeros. Toggle cells and watch recommendations surface.

From basic MF to implicit MF

Three edits turn one model into the other

The factorization r̂ᵤᵢ = uᵤᵀvᵢ never changes. Only what we fit, and how much we trust it, changes.

BASIC (explicit)
min Σ(u,i) ∈ observed ( rui − uᵤᵀvᵢ )² + λ(‖U‖²+‖V‖²)
  • target = the star rating, 1…5
  • sum only over cells we actually saw
  • blanks are simply skipped
IMPLICIT (iALS)
min Σall (u,i) ∈ U×I cui ( pui − uᵤᵀvᵢ )² + λ(‖U‖²+‖V‖²)
  • preference pui = 1 if interacted, else 0
  • confidence cui = 1+α if interacted, else α
  • sum over every pair — blanks are weak zeros, not skips
Why it must change: a "0" is missing data, not a bad review. Skipping zeros (basic MF) leaves the model free to predict 1 everywhere; treating them as hard zeros calls every unseen film bad. iALS threads the needle — zeros are pulled toward 0 but with low confidence α, so a strong pattern can still lift an unseen item above them. That lift is the recommendation.

03 — two domains, one set of users

Learn together, without pooling the data.

Two companies share customers but not catalogs. Each runs its own ALS locally; a consensus vector Z pulls their shared-user factors together round by round. Nobody ships raw histories — only the quantity Xᵢ + Uᵢ. Watch the two clouds meet, then read the fine print.

Derivation

From a constraint to the three ADMM steps

The whole method falls out of one idea: force the shared users' factors to agree, then relax that constraint with a Lagrangian.

Step 1 · the constrained problem. Each domain minimises its own loss Fᵢ, but shared users must land on one vector:
min{Xᵢ},{Yᵢ},Z   Σᵢ Fᵢ(Xᵢ, Yᵢ) + g(Z)    subject to    Xᵢ − Z = 0   ∀i
Z is the consensus copy; g is an optional regulariser on it.
Step 2 · the Lagrangian. Attach a dual matrix Λᵢ to each constraint:
L = Σᵢ [ Fᵢ + ⟨Λᵢ , Xᵢ − Z⟩ ] + g(Z)   — but this saddle point converges poorly on its own.
Step 3 · augment it. Add a quadratic penalty ρ on constraint violation — convexifies the subproblems and gives a step size:
Lρ = Σᵢ [ Fᵢ + ⟨Λᵢ , Xᵢ − Z⟩ + (ρ/2)‖Xᵢ − Z‖²F ] + g(Z)
Step 4 · scale the dual. Put Uᵢ = Λᵢ/ρ and complete the square. The linear and quadratic terms merge:
⟨Λᵢ, Xᵢ−Z⟩ + (ρ/2)‖Xᵢ−Z‖²  =  (ρ/2)‖Xᵢ − Z + Uᵢ‖² − (ρ/2)‖Uᵢ‖²
the last term is constant in Xᵢ, so it drops out of the minimisation →
Lρ = Σᵢ [ Fᵢ(Xᵢ,Yᵢ) + (ρ/2)‖Xᵢ − Z + Uᵢ‖²F ] + g(Z)
Step 5 · minimise alternately → ADMM. Fix two blocks, solve the third:
• over Xᵢ,Yᵢ  →  X-step (a local ALS with a ρ-pull toward Z − Uᵢ)
• over Z  →  Z-step, Z ← proxg,1/ρN( ⟨X⟩ + ⟨U⟩ )
• gradient ascent on the dual  →  Uᵢ ← Uᵢ + (Xᵢ − Z)
Notice what the X-step became: argmin Fᵢ + (ρ/2)‖Xᵢ − H‖² with H = Z − Uᵢ. That is exactly the proximal term in the generic subproblem on station 04 — which is why CDIMF needs no new solver, only a ρI added to the Gram matrix and a ρH added to the right-hand side.

Derivation

From a hard constraint to the three ADMM steps

Consensus is a constraint. The Augmented Lagrangian is how we turn it into something each domain can solve on its own.

Step 0 · the constrained problem. Each domain keeps its own factors, but shared users must agree:
min{Xᵢ},{Yᵢ},Z   Σᵢ Fᵢ(Xᵢ, Yᵢ) + g(Z)    subject to   Xᵢ − Z = 0   ∀i
Step 1 · attach a multiplier. Price the violation with a dual variable Λᵢ:
L = Σᵢ Fᵢ + g(Z) + Σᵢ ⟨ Λᵢ , Xᵢ − Z ⟩   — exact, but the saddle point is hard to reach
Step 2 · augment it. Add a quadratic penalty; it makes each subproblem strongly convex and stabilises the iteration:
Lρ = Σᵢ Fᵢ + g(Z) + Σᵢ ⟨ Λᵢ , Xᵢ − Z ⟩ + (ρ/2) Σᵢ ‖Xᵢ − Z‖²F
Step 3 · scale the dual, complete the square. Set Uᵢ = Λᵢ / ρ. The two coupling terms merge:
⟨Λᵢ , Xᵢ−Z⟩ + (ρ/2)‖Xᵢ−Z‖²  =  (ρ/2)‖Xᵢ − Z + Uᵢ‖² − (ρ/2)‖Uᵢ‖²  (last term is constant in Xᵢ)
⇒   Lρ = Σᵢ [ Fᵢ(Xᵢ,Yᵢ) + (ρ/2)‖Xᵢ − Z + Uᵢ‖²F ] + g(Z)
Step 4 · minimise alternately → the algorithm.
X-step (each domain, alone)   Xᵢk+1, Yᵢk+1 ← argmin Fᵢ + (ρ/2)‖Xᵢ − Zk + Uᵢk‖²
Z-step (aggregate)              Zk+1 ← proxg, 1/ρN( ⟨X⟩k+1 + ⟨U⟩k )
dual ascent                  Uᵢk+1 ← Uᵢk + ( Xᵢk+1 − Zk+1 )
Read the X-step carefully: it is an ordinary ALS objective plus a pull toward Z − Uᵢ. That is the whole trick — the constraint never leaves the local solver, it just becomes one extra quadratic term. Uᵢ accumulates the running disagreement, so a domain that keeps drifting is pulled back harder each round. Station 04 shows this costs only a ρI added to the d×d matrix.
X-step Xᵢ ← argmin Fᵢ(Xᵢ,Yᵢ) + (ρ/2)‖Xᵢ − Z + Uᵢ‖²  ·  Z-step Z ← proxg( ⟨X⟩ + ⟨U⟩ )  ·  dual Uᵢ ← Uᵢ + (Xᵢ − Z)

The fine print

What actually crosses the wire

CDIMF claims the broadcast Xᵢ + Uᵢ conceals the factors. It doesn't.

U⁰ = 0  ⇒  the first message is X¹ᵢ, in the clear.
Z is the public mean of the messages  ⇒  anyone can recompute it.
The dual update is deterministic  ⇒  Uᵢk+1 = wᵢk − Zk+1, then Xᵢk+1 = wᵢk − Uᵢk — by induction, every factor, every round.
So the mask offers no secrecy against a passive observer. Stations 05 and 06 fix exactly this: secure aggregation hides the individual messages, and differential privacy bounds what the shared factors reveal.

04 — the new update rule

Turn up d for richer factors — the cost stays friendly.

Both panels run the same latent dimension as it climbs automatically. The standard all-pairs cost is dominated by the catalog and balloons; the proposed Gram-trick update grows only like d² and each factor solves independently, in parallel.

Derivation

Where the update rule comes from

Every CDIMF factor update — user or item, local or shared — is one instance of the same convex subproblem.

Step 0 · the subproblem.
f(A) = ½‖C ∘ (P − ABᵀ)‖²F + (λ/2)‖RA‖²F + (ρ/2)‖A − H‖²F   ← the ρ term is the consensus pull from station 03
Step 1 · set the gradient to zero. R is diagonal, so
∂f/∂A = −[ C∘2 ∘ (P − ABᵀ) ] B + λR²A + ρ(A − H)  =  0
Step 2 · the rows decouple. Rows of A interact only through the shared B, so take one row a (with p, c the matching rows of P, C and r its scalar reg. weight):
−[ (p − aBᵀ) ∘ c∘2 ] B + λr²a + ρ(a − h) = 0
Step 3 · collect the terms in a. Using (aBᵀ)diag(c∘2)B = a·Bᵀdiag(c∘2)B:
a·Bᵀdiag(c∘2)B + λr²a + ρa  =  (p ∘ c∘2)B + ρh
Step 4 · transpose → the new form.
[ Bᵀ diag(c∘2) B + (ρ + λr²) Id ] a  =  Bᵀ(p ∘ c∘2) + ρh
a d×d system per factor · solved independently · ρ=0, h=0 recovers plain iALS
Read the boxed line as a cost statement: the matrix to invert is d×d — the catalog size never enters it. The all-pairs term hides inside Bᵀdiag(c∘2)B, which collapses to a single reusable Gram matrix α·BᵀB computed once per epoch. That is exactly why the curves below diverge.
latent dimension d = 1

Standard approach

Comparing every user against every item, per dimension — O(d·|U|·|I|).

Proposed method

One reused d×d Gram matrix — O(d²·(|U|+|I|) + d·|S|), solved per factor independently.

05 — fixing the Z-step

Aggregate the messages. Never see one.

Station 03 showed the leak: the broadcast wᵢ = Xᵢ + Uᵢ can be un-masked, because Z is the public mean and the dual update is deterministic. Secure aggregation gives the Z-step exactly the sum it needs — Σᵢ wᵢ — while every individual wᵢ stays hidden, even from the coordinator.

Z-step needs only  ⟨X⟩ + ⟨U⟩ = (1/N) Σᵢ wᵢ  — a sum, never the terms.  ·  each pair {i,j} shares a secret mask sij; i adds it, j subtracts it.

The masking round

Four domains, each holding one shared user's message wᵢ = Xᵢ + Uᵢ ∈ ℝ².

Each domain holds its own wᵢ. Step through the round.
latent space — where the messages live
true wᵢ = Xᵢ+Uᵢ masked broadcast w̃ᵢ recovered Z

Who sends what to whom

Masks live on the edges and cancel in pairs. The coordinator only ever adds up.

Why it plugs into CDIMF unchanged

The Z-step is an average — and averages are all this protocol reveals

Σᵢ w̃ᵢ = Σᵢ [ wᵢ + Σj>i sij − Σj<i sij ]
     = Σᵢ wᵢ + Σi<j ( sij − sij ) = Σᵢ wᵢ
  • The ALS solver, ρ, and the dual update are untouched.
  • Each w̃ᵢ alone is pseudo-uniform → reveals nothing.
  • Dropouts handled by t-of-N secret-sharing the seeds.
One honest caveat: with only N = 2 domains — the dual-domain setting CDIMF is evaluated in — a participant who knows its own w₁ can subtract it from the revealed sum and recover w₂ exactly. Secure aggregation needs N ≥ 3, or it must be paired with the noise from station 06. That is why the two defenses are complementary, not alternatives.

06 — bounding what the factors reveal

Add just enough noise to make one user deniable.

Secure aggregation hides the message in transit. But the converged factors are shared by design (Xᵢ → Z), and embeddings leak attributes and memberships. Differential privacy bounds that leak with a formal guarantee — and it slots into the CDIMF loop as three extra lines.

Implementation

Where DP goes in the CDIMF round

Only the X-step's outgoing message changes. The ALS solver, ρ, Z-step, and dual update are untouched.

for round k = 1 … K, at each domain i:
  1. Xᵢ, Yᵢ ← local ALS solve  (unchanged)
  2. clip every shared-user row:   x̄ᵤ ← xᵤ · min( 1 , B / ‖xᵤ‖₂ )  ← bounds sensitivity
  3. form the message   wᵢ = X̄ᵢ + Uᵢ
  4. add noise:   w̃ᵢ = wᵢ + 𝒩(0, σ²I),   σ = Δ₂·√(2 ln(1.25/δ)) / εround
  5. secure-aggregate the w̃ᵢ  (station 05)
  6. Z ← proxg( ⟨X̃⟩ + ⟨U⟩ ) ;   Uᵢ ← Uᵢ + (Xᵢ − Z)  (unchanged)
after K rounds: compose the per-round budgets → total ε
Why clipping comes first

Noise must scale with how much one user can move the output. Un-clipped, a single row is unbounded, so no finite σ suffices. Clipping to B forces the L₂ sensitivity Δ₂ ≤ 2B (replace-one-user) or ≤ B (add/remove).

Why composition matters

A message goes out every round, so privacy loss accumulates. Rényi-DP composes tightly: ε(δ) = minα>1 [ K·α·Δ₂²/(2σ²) + ln(1/δ)/(α−1) ]. Fewer rounds (larger aggregation period) ⇒ less noise for the same total ε.

Indistinguishability

Two neighbouring datasets (differ by one user) → two output distributions. DP forces them to overlap.

smaller ε = more noise = stronger privacy · more rounds = more total leakage

The trade-off

A jittering released value, the attacker's odds, and the price in accuracy.

DP and secure aggregation defend different things: aggregation stops anyone from reading a single message; DP stops the released aggregate and final factors from betraying whether any one user was in the data at all.
